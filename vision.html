🧠 Product Vision

The Fast + Slow Brain

AI conversations use fast thinking to align context in real-time. Deep execution uses slow thinking to optimize outcomes. Together, they form the future of agent architecture.

Cognitive Architecture

Two Systems, One Agent

Inspired by Kahneman's dual-process theory: System 1 handles fast, intuitive responses while System 2 handles slow, deliberate reasoning.

System 1 — Fast Thinking

Voice AI

OpenAI Realtime API

Conversational, intuitive, immediate. System 1 handles real-time dialogue — answering questions, guiding discussions, and aligning context through natural voice interaction.

Context Bridge

Three Mechanisms, Zero Gap

System 1 and System 2 have separate context windows. These three bridges keep them in sync.

📋

Meeting Prep Brief

Before each meeting, System 2 generates a structured brief and injects it into System 1's instructions. Includes talking points, file paths, and expected Q&A.

S2 generates → S1 receives
🔄

ContextSync

Continuously aggregates context from MEMORY.md, pinned files, and session notes. Generates tiered briefs — compact for Voice, detailed for Computer Use.

S1 reads ↔ S2 writes
⚡

Dynamic Push

During live meetings, System 2 pushes context updates in real-time. Live notes, completed tasks, and new requirements are injected into Voice AI's active session.

S2 pushes → S1 updates live
